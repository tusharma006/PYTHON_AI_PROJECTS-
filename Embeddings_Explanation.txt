Embeddings and Vector Database for Song Recommendation

In this project, every song from the JSON dataset is converted into a LangChain Document. The notebook creates a page_content field by combining the song name, artist, genre, mood, tempo, language, release year, keywords, and description into a single natural language text. This approach is used because embedding models are trained on natural language rather than Python dictionary syntax, allowing them to capture the overall meaning of each song more effectively. The original song information is also stored as metadata so that individual fields can be displayed after retrieval.

The project uses the HuggingFace embedding model `sentence-transformers/all-MiniLM-L6-v2` to generate embeddings for the page_content of every song. These embeddings are stored in a Chroma vector database using `Chroma.from_documents()` with the `persist_directory="chroma_db"` option so the vector database can be reused instead of being recreated every time the notebook runs.

When a user enters a query, the same embedding model converts the query into a vector. ChromaDB then compares the query vector with the stored song vectors using `similarity_search_with_score()` and returns the five most similar songs. The notebook displays the song details from the metadata along with the similarity score. Lower scores indicate that the song embedding is closer to the query embedding, making it a more relevant recommendation. By combining semantic embeddings with vector similarity search, the system recommends songs based on meaning rather than exact keyword matches.
